Reference Semantics

Family mask rule

- Non-masked modes move full bytes by the family index rule.
- Masked modes use the same index rule, but only move masked bits.
- For every destination byte:
- result = (source_byte & ~mask) | (moved_byte & mask)

Reverse meaning
- Reverse changes order by reading from the other end of a region.
- It is symmetric by design: applying again restores original order.

Reverse forms
- base: Reverse complete byte order once; first becomes last, with symmetry.
- block: Reverse order of whole blocks; bytes inside each block stay.
- block byte: Within each block, reverse byte order; keep block positions unchanged.

Rotation family files
- /Rotation/RotateCipher.hpp
- /Rotation/RotateBlockCipher.hpp
- /Rotation/RotateBlockByteCipher.hpp
- /Rotation/RotateMaskCipher.hpp
- /Rotation/RotateMaskBlockCipher.hpp
- /Rotation/RotateMaskByteBlockCipher.hpp

Rotation meaning
- Rotation shifts positions by a signed amount and wraps at edges.
- The active region defines where wrap happens: full data or block window.

Rotation forms
- base: Rotate whole byte region by shift amount; wrapped indices preserve size.
- block: Rotate block positions across region; each block contents move together.
- block byte: Rotate bytes inside every block; block positions remain fixed globally.

Splint meaning
- Splint divides data into front and back spans, then interleaves them.
- It is a deterministic split-merge shuffle over bytes or blocks.

Splint forms
- base: Interleave front and back byte halves into alternating output sequence.
- block: Interleave front and back block halves into alternating block stream.
- block byte: Per block, interleave front and back byte halves locally.

Weave family files
- /Weave/WeaveCipher.hpp
- /Weave/WeaveBlockCipher.hpp
- /Weave/WeaveByteBlockCipher.hpp
- /Weave/WeaveMaskCipher.hpp
- /Weave/WeaveMaskBlockCipher.hpp

Weave meaning
- Weave walks from front and back, swapping by count and stride controls.
- It creates sparse crossing exchanges instead of full contiguous permutations.

Weave forms
- base: Swap front/back byte positions repeatedly using count and stride steps.
- block: Swap front/back blocks repeatedly using count and stride block steps.
- block byte: Inside each block, weave bytes with local front/back stride swaps.

Ripple family files

- /Ripple/RippleCipher.hpp
- /Ripple/RippleBlockCipher.hpp
- /Ripple/RippleMaskCipher.hpp
- /Ripple/RippleMaskBlockCipher.hpp

Ripple meaning
- Ripple runs neighbor swaps in passes, like odd-even transposition waves.
- More passes increase mixing while keeping operations very cache friendly.

Ripple forms
- base: Swap adjacent bytes in repeating passes; odd and even phases.
- block: Swap adjacent blocks in repeating passes; odd and even phases.

Proposed new behavior: Jump

Jump meaning
- Jump permutes by stepping through indices with a fixed stride K.
- Choose K coprime with region length to get a full-cycle walk.

Jump forms
- base: Reorder bytes by fixed stride walk over full index cycle.
- block: Reorder blocks by fixed stride walk over block index cycle.
- block byte: Per block, reorder bytes by fixed stride local index walk.

SwapGrid meaning
- SwapGrid rolls bytes into a square tile, runs pair swaps, then unrolls.
- Tile size is fixed by variant: 16 bytes (4x4) or 64 bytes (8x8).

SwapGrid variants
- naming: SwapGrid[swap][unroll][size]
- first letter (swap): V = swap row pairs, H = swap column pairs
- second letter (unroll): H = row-major output, V = column-major output
- size: 16 or 64 tile bytes

SwapGrid class set

- SwapGridVV64, SwapGridVH64, SwapGridHV64, SwapGridHH64

SpiralGrid meaning
- SpiralGrid rotates each ring of a square tile by pAmount.
- Unseal uses the same operation with -pAmount.

SpiralGrid variants

- unroll H: row-major ring indexing
- unroll V: column-major ring indexing
- size: 16 or 64 tile bytes

Invert family files

- /Invert/InvertCipher.hpp
- /Invert/InvertMaskCipher.hpp

Invert meaning
- Invert flips bits in place, no index movement.
- It is symmetric by design: applying again restores original bytes.

Invert forms
- base: Invert every bit in every byte.
- mask: Invert only masked bits; keep unmasked bits unchanged.

Common rules

- Data is an ordered byte list.
- Every cipher call uses a data length that is a multiple of 192.
- 192 remains the rule. It is a clean multiple of 48, 32, 24, 16, 12, and 8.
- Every execution point must fail if length % 192 != 0.
- Benchmarking is only valid in Release mode.
- Benchmarking always uses max optimization for the active toolchain.
- The benchmark script must clean, rebuild Release, then run benchmarks.
- A block mode splits data into blocks of size B.
- A byte-block mode keeps the same blocks, then works on bytes inside each block.
- A mask mode moves only masked bits.
- Unmasked bits stay in place.

Mask rule

Given:
- byte x
- mask m
- antimask a = ~m

Split:
- masked = x & m
- base = x & a

After the operation:
- result = base | moved_masked

Rotate

Byte mode:
- rotate the byte list by shift s
- result[i] = source[(i + r) mod N]
- r = normalized shift

Mask byte mode:
- rotate only masked bits
- result[i] = (source[i] & ~m) | (source[(i + r) mod N] & m)

Block mode:
- split data into blocks of size B
- for each full block:
- rotate bytes inside that block by shift s

Mask block mode:
- split data into blocks of size B
- for each full block:
- result_block[i] = (source_block[i] & ~m) | (source_block[(i + r) mod B] & m)

Splint

Byte mode:
- split byte list into front half and back half
- output alternates:
- front[0], back[0], front[1], back[1], ...
- if one side is longer, append the rest

Byte decrypt:
- take even output indices as front
- take odd output indices as back
- write front, then back

Mask byte mode:
- same byte position rule as Splint
- move only masked bits
- keep unmasked bits in original byte positions

Block mode:
- split data into blocks of size B
- use only full blocks for the block permutation
- split full blocks into front half and back half
- output blocks:
- front[0], back[0], front[1], back[1], ...
- partial last block, if any, stays at the end

Block decrypt:
- from full blocks:
- even block indices go to front
- odd block indices go to back
- write front, then back
- partial last block, if any, stays at the end

Mask block mode:
- do the same block permutation as Splint block mode
- then for each byte position:
- result[i] = (source[i] & ~m) | (permuted[i] & m)

Byte-block mode:
- split data into blocks of size B
- for each block:
- apply Splint to bytes inside the block

Mask byte-block mode:
- for each block:
- apply Splint to bytes inside the block
- then for each byte position in the full data:
- result[i] = (source[i] & ~m) | (permuted[i] & m)

Weave

Byte mode:
- front = 0
- back = N - 1
- swap count times
- then move front by frontStride
- then move back by backStride
- repeat until front >= back

Mask byte mode:
- same front and back walk
- at each swap pair:
- swap only masked bits
- keep unmasked bits in place

Block mode:
- split data into blocks of size B
- use only full blocks for the block walk
- front = first full block
- back = last full block
- swap whole blocks count times
- then move front by frontStride
- then move back by backStride
- repeat until front >= back
- partial last block, if any, stays at the end

Mask block mode:
- same block walk as Weave block mode
- when swapping two blocks:
- for each byte j in the block:
- left[j] = (left[j] & ~m) | (right_old[j] & m)
- right[j] = (right[j] & ~m) | (left_old[j] & m)

Byte-block mode:
- split data into blocks of size B
- for each block:
- do the Weave byte walk inside that block

Mask byte-block mode:
- for each block:
- do the Weave byte walk inside that block
- each byte swap moves only masked bits
- unmasked bits stay in place
