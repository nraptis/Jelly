Reference Semantics

Common rules

- Data is an ordered byte list.
- Every cipher call uses a data length that is a multiple of 192.
- 192 remains the rule. It is a clean multiple of 48, 32, 24, 16, 12, and 8.
- Every execution point must fail if length % 192 != 0.
- Benchmarking is only valid in Release mode.
- Benchmarking always uses max optimization for the active toolchain.
- The benchmark script must clean, rebuild Release, then run benchmarks.
- A block mode splits data into blocks of size B.
- A byte-block mode keeps the same blocks, then works on bytes inside each block.
- A mask mode moves only masked bits.
- Unmasked bits stay in place.

Mask rule

Given:
- byte x
- mask m
- antimask a = ~m

Split:
- masked = x & m
- base = x & a

After the operation:
- result = base | moved_masked

Rotate

Byte mode:
- rotate the byte list by shift s
- result[i] = source[(i + r) mod N]
- r = normalized shift

Mask byte mode:
- rotate only masked bits
- result[i] = (source[i] & ~m) | (source[(i + r) mod N] & m)

Block mode:
- split data into blocks of size B
- for each full block:
- rotate bytes inside that block by shift s

Mask block mode:
- split data into blocks of size B
- for each full block:
- result_block[i] = (source_block[i] & ~m) | (source_block[(i + r) mod B] & m)

Splint

Byte mode:
- split byte list into front half and back half
- output alternates:
- front[0], back[0], front[1], back[1], ...
- if one side is longer, append the rest

Byte decrypt:
- take even output indices as front
- take odd output indices as back
- write front, then back

Mask byte mode:
- same byte position rule as Splint
- move only masked bits
- keep unmasked bits in original byte positions

Block mode:
- split data into blocks of size B
- use only full blocks for the block permutation
- split full blocks into front half and back half
- output blocks:
- front[0], back[0], front[1], back[1], ...
- partial last block, if any, stays at the end

Block decrypt:
- from full blocks:
- even block indices go to front
- odd block indices go to back
- write front, then back
- partial last block, if any, stays at the end

Mask block mode:
- do the same block permutation as Splint block mode
- then for each byte position:
- result[i] = (source[i] & ~m) | (permuted[i] & m)

Byte-block mode:
- split data into blocks of size B
- for each block:
- apply Splint to bytes inside the block

Mask byte-block mode:
- for each block:
- apply Splint to bytes inside the block
- then for each byte position in the full data:
- result[i] = (source[i] & ~m) | (permuted[i] & m)

Weave

Byte mode:
- front = 0
- back = N - 1
- swap count times
- then move front by frontStride
- then move back by backStride
- repeat until front >= back

Mask byte mode:
- same front and back walk
- at each swap pair:
- swap only masked bits
- keep unmasked bits in place

Block mode:
- split data into blocks of size B
- use only full blocks for the block walk
- front = first full block
- back = last full block
- swap whole blocks count times
- then move front by frontStride
- then move back by backStride
- repeat until front >= back
- partial last block, if any, stays at the end

Mask block mode:
- same block walk as Weave block mode
- when swapping two blocks:
- for each byte j in the block:
- left[j] = (left[j] & ~m) | (right_old[j] & m)
- right[j] = (right[j] & ~m) | (left_old[j] & m)

Byte-block mode:
- split data into blocks of size B
- for each block:
- do the Weave byte walk inside that block

Mask byte-block mode:
- for each block:
- do the Weave byte walk inside that block
- each byte swap moves only masked bits
- unmasked bits stay in place
